# 專案名稱：個人記帳簿系統

## 1. 專案概述
- **背景與動機**：現代人常因為沒有記帳習慣而導致不知不覺花費過多，月底經常成為「月光族」。為了解決這個問題，希望能有一個簡單易用的工具，幫助使用者快速記錄日常收支，掌握自己的財務狀況。
- **目標用戶**：想管理個人財務的大學生或上班族。
- **核心價值主張**：提供一個直覺、自動計算餘額且能圖表化呈現支出的記帳工具，幫助使用者輕鬆建立記帳習慣。

## 2. 功能需求
1. **新增收入與支出記錄**
   - 作為 **使用者**，我希望 **能輸入金額、日期、分類與備註來新增收支記錄**，以便 **完整記錄每一筆交易**。
2. **顯示目前的總餘額**
   - 作為 **使用者**，我希望 **在首頁直接看到目前的總餘額**，以便 **隨時掌握可用資金**。
3. **依照日期查看歷史記錄**
   - 作為 **使用者**，我希望 **能依日期排序瀏覽歷史收支記錄**，以便 **回顧過去的消費明細**。
4. **分類管理**
   - 作為 **使用者**，我希望 **能在新增記錄時選擇不同的分類（如飲食、交通、娛樂）**，以便 **歸納我的花費項目**。
5. **顯示本月支出圓餅圖**
   - 作為 **使用者**，我希望 **能看到本月各分類支出的圓餅圖比例**，以便 **直觀了解我的主要開銷落在哪個類別**。

## 3. 非功能需求
- **技術限制**：後端採用 Flask 框架，前端視圖使用 Jinja2 模板引擎，資料庫使用 SQLite。
- **效能與安全考量**：
  - 系統需防範基本的 SQL Injection 與 XSS 攻擊。
  - 頁面載入時間應力求快速。
  - 確保使用者輸入資料的驗證與錯誤處理，避免因格式錯誤導致系統崩潰。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 新增收入與支出記錄。
  - 顯示目前的總餘額。
  - 依照日期查看歷史記錄。
  - 基本的收支分類選擇。
- **Should Have (應該有)**：
  - 顯示本月支出圓餅圖，視覺化呈現花費比例。
- **Nice to Have (可以有)**：
  - 支援客製化新增與刪除分類。
  - 匯出收支記錄為 CSV 檔案。
  - 深色模式支援。
